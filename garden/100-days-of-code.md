100 days of code is a challenge where you have to code for a 100 days, and write a few lines about it each time. The days do not have to be consecutive, and you can code for as long as you want each day.

# Goals

There is a few things I want to achieve with this challenge :

-   Learn the Godot game engine with tutorials
-   Polish / finish my website
-   Document my progress and my learning process
-   Create an article for each day of the challenge
-   Create a video at the end of the challenge, that will summarize my progress.
-   Ultimately, create my dream game and be proud of it

It is important to record my progress, as it will allow me to create the video at the end. Timelapses of me coding are not very interesting, so I should focus on the playtest and testing process of the game.

In order to manage the videos and my computer's storage, I will record the videos and upload them to youtube at the end of the day, and delete them from my computer. If I have many videos, I can sort them using playlists for each day.

# The game

As I said, one of my goals during this challenge is to create my dream game. I have a few ideas, but I will not reveal everything about it yet. I will reveal more about it as I progress through the challenge.

All I will say for now is that it will be a rogue-lite game with hopefully a lot of replayability. It might be a mix between Isaac, Enter the Gungeon and Nuclear Throne. I also take inspiration from other games such as Kingdom new lands.

My goal is to be proud of it and have at least a few person playing it and hopefully enjoying it.
